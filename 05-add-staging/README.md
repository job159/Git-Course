# 05 把檔案加入暫存區：`git add`

> `git add` 不是上傳，它是告訴 Git：「這些內容我要放進下一次提交。」

## 今天只學這些指令

| 指令 | 用途 |
| --- | --- |
| `git add <file>` | 精準加入某一個檔案 |
| `git add .` | 把目前資料夾底下所有變更一起加入暫存區 |

## 最推薦的新手用法

```bash
git status
git add README.md
git status
```

## 快速全部加入

```bash
git add .
git status
```

## `git add` 真正加入的是什麼

| 狀況 | 會不會進暫存區 |
| --- | --- |
| 新增的檔案 | 會 |
| 已修改的檔案 | 會 |
| 已刪除的檔案 | 通常也會被記錄到 |

## 什麼時候用 `git add <file>`，什麼時候用 `git add .`

| 用法 | 適合情境 | 優點 | 注意 |
| --- | --- | --- | --- |
| `git add <file>` | 你想精準提交 | 乾淨、可控 | 要多打一點字 |
| `git add .` | 你已確認所有變更都該提交 | 快 | 容易把不想交的東西一起放進去 |

## 很重要的觀念

| 觀念 | 說明 |
| --- | --- |
| `add` 加的是「目前內容快照」 | 不是單純把檔名加進去 |
| 先 `add` 再修改 | 後面修改的內容不會自動跟進暫存區 |
| 可以分批 `add` | 這樣一個提交只做一件事會更乾淨 |

## 一個常見情境

1. 你修改了 `README.md` 與 `app.js`。
2. 這次只想先提交 `README.md`。
3. 就只執行下面這段：

```bash
git add README.md
git commit -m "Update README wording"
```

## 本章你要記住

1. `git add` 是把內容放進暫存區，不是直接形成版本。
2. 新手最穩的用法是先 `git status`，再 `git add <file>`。
3. `git add .` 很快，但要先確認沒有多餘變更。
