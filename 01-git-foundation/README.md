# 01 Git 核心觀念：先知道自己在做什麼

> 如果只記住一句話：Git 不是雲端硬碟，它是「幫你記住版本歷史」的工具。

## 本章目標

| 你要理解的事 | 說明 |
| --- | --- |
| Working Tree | 你目前正在編輯的檔案區域 |
| Staging Area | 你準備要提交的內容暫存區 |
| Repository | Git 真正記錄歷史版本的地方 |
| Commit | 一個有名稱、有時間點的版本快照 |

## 今天只學這一個指令

| 指令 | 用途 | 什麼時候用 |
| --- | --- | --- |
| `git --version` | 確認 Git 是否已安裝 | 第一次開始前 |確認版本

## 先確認 Git 能不能用

```bash
git --version
```

## Git 的四個核心區域

| 區域 | 你可以把它想成 | 常見動作 |
| --- | --- | --- |
| Working Tree | 你的桌面草稿 | 修改檔案 |
| Staging Area | 準備送出的打包區 | `git add` |
| Local Repository | 你的本機版本倉庫 | `git commit` |
| Remote Repository | 團隊共用的遠端倉庫 | `git push`、`git pull` |

## 你之後最常做的基本流程

| 步驟 | 指令 | 目的 |
| --- | --- | --- |
| 1 | `git status` | 看現在有哪些變更 |
| 2 | `git add <file>` | 把想提交的變更放進暫存區 |
| 3 | `git commit -m "訊息"` | 建立版本點 |
| 4 | `git push` | 推到遠端和團隊同步 |

## 先別急著背一堆指令

| 先學會什麼 | 原因 |
| --- | --- |
| `status` | 這是最安全的觀察指令 |
| `add` | 你要決定哪一些修改要進本次提交 |
| `commit` | Git 的核心價值就是建立可追蹤的版本 |
| `log` | 你需要有能力回頭看歷史 |

## 新手最常誤解的地方

| 誤解 | 正確理解 |
| --- | --- |
| 存檔就等於 Git 記住了 | 不是，Git 要到 `commit` 才真的形成版本 |
| `add` 是上傳 | 不是，`add` 只是放進暫存區 |
| Git = GitHub | 不是，Git 是工具；GitHub 是託管平台 |
| 每次都要學很難的指令 | 不是，先把高頻核心指令練熟最重要 |

## 本章你要記住

1. Git 的核心不是「備份」，而是「版本管理」。
2. 先學會看狀態，再學提交，效率會高很多。
3. 只靠 20% 到 40% 的高頻指令，就能完成大多數日常工作。
