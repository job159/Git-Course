# 19 安全回退與救援：`revert`、`reflog`

> 新手最需要的不是最猛的破壞性指令，而是最安全的回退與救援方式。

## 今天只學這些指令

| 指令 | 用途 |
| --- | --- |
| `git revert <commit>` | 用一筆新的反向提交撤銷舊提交 |
| `git reflog` | 查看 HEAD 曾經去過哪裡，拿來救援很強 |

## 安全撤銷一筆已提交的變更

```bash
git log --oneline
git revert abc1234
```

## 當你找不到剛剛的狀態

```bash
git reflog
```

## 為什麼新手先學 `revert`

| 原因 | 說明 |
| --- | --- |
| 安全 | 它不會粗暴刪掉歷史，而是新增一筆「反向操作」 |
| 適合共享分支 | 在已推送歷史上也比較穩 |

## `reflog` 能幫你什麼

| 場景 | 幫助 |
| --- | --- |
| 剛剛切分支、回退、amend 後找不到舊位置 | 看 `reflog` 找回來 |
| 不小心跑到陌生狀態 | 看看 HEAD 是怎麼移動的 |

## 現階段先記住的安全原則

| 原則 | 說明 |
| --- | --- |
| 已分享的歷史，優先考慮 `revert` | 比較不會傷到別人 |
| 找不到剛剛那筆提交，先看 `reflog` | 先別慌，通常找得回來 |
| 不急著學危險重寫歷史指令 | 先把安全解法學熟 |

## 本章你要記住

1. `git revert` 是安全撤銷已提交變更的第一選擇。
2. `git reflog` 是救援地圖，超級實用。
3. 卡住時先查歷史，再決定下一步，不要亂重置。
