# 13 合併分支：`git merge`

> 功能做完後，真正把它接回主線的動作，就是 `merge`。

## 今天只學這些指令

| 指令 | 用途 |
| --- | --- |
| `git merge <branch>` | 把指定分支合併進目前分支 |
| `git branch -d <branch>` | 合併完成後刪除本地功能分支 |

## 最常見的合併流程

```bash
git switch main
git merge feature/login-page
git branch -d feature/login-page
```

## 你一定要記住的規則

| 規則 | 說明 |
| --- | --- |
| 你站在哪裡，就會把別人合進來 | 在 `main` 上執行 `merge feature/x`，表示把 `feature/x` 合進 `main` |
| 先切到目標分支再 merge | 這是最不容易搞錯的做法 |

## 合併後刪分支是好習慣

| 好處 | 說明 |
| --- | --- |
| 分支列表更乾淨 | 已完成的功能不用一直留著 |
| 降低混亂 | 不會把老分支誤認成還在進行中 |

## 遇到衝突怎麼辦

| 步驟 | 做法 |
| --- | --- |
| 1 | 打開衝突檔，手動選擇要保留的內容 |
| 2 | 存檔後執行 `git add <file>` |
| 3 | 完成合併提交 |

## 衝突解完後可直接複製

```bash
git add app.js
git commit
```

## 本章你要記住

1. `merge` 幾乎一定要在目標分支上做。
2. 合併完成後，刪掉本地功能分支通常是好習慣。
3. 衝突不可怕，重點是先修內容，再 `add`，再完成提交。
